## 10. Multilingual Support (Internationalization)  
    Drive Indicator AI uses an internationalization system that manages all UI text in JSON files.  
    This allows new languages ​​to be added without recompiling the app,  
    providing a flexible design that allows users to instantly switch UI languages.  

### 10.1 Overall Structure of Multilingual Support  
    Multilingual support consists of the following three elements :  
      1. Language Files (JSON)  
      2. LangManager (Translation Management Class)  
      3. Application in the UI (SettingsForm / TrayIconManager, etc.)  
      ───────────────────────────────────────────────────  
      lang_xx.json → LangManager → Application to the UI  
      ───────────────────────────────────────────────────

    LangManager reads this file and performs the following mutual conversions :  
      • Language code           → Language name  
      • Language code           → Language name (English)  
      • Language name           → Language code  
      • Language name (English) → Language code  
      • Language name           → Language name (English)  

### 10.4 Role of LangManager  
    LangManager is the central class for multilingual support.  

      LangManager  
        ├─ Load(languageCode)  
        ├─ T(key)  
        ├─ LoadLanguagesJson()  
        ├─ GetNameToCode(languageName)  
        ├─ GetEnameToCode(languageEname)  
        ├─ GetName(languageCode)  
        ├─ GetEname(languageCode)  
        └─ GetNameToEname(languageName)  

#### 10.4.1 Load(languageCode)  
    • Loads lang_xx.json  
    • Stores in a Dictionary<string, string>  
    • Builds a translation table  

#### 10.4.2 T(key)  
    Method for retrieving translation strings.  
      ──────────────────────────────────────────────────  
      string text = Lang.T("Settings");  
      ──────────────────────────────────────────────────  
    If the key does not exist, the key name is returned as is, so the UI will not break even if a translation is missing.  

### 10.5 Application in UI  
    UI components use LangManager to update text.  

#### 10.5.1 SettingsForm Example  
    ───────────────────────────────────────────────────  
    labelSettings.Text = Lang.T("labelSettingsText");  
    BtnSave.Text = Lang.T("BtnSaveText");  
    ───────────────────────────────────────────────────  

#### 10.5.2 TrayIconManager Example  
    ───────────────────────────────────────────────────  
    menu.Items.Settings.Text = Lang.T("MenuSettings");  
    menu.Items.Quit.Text = Lang.T("MenuQuit");  
    ───────────────────────────────────────────────────  

#### 10.5.3 Behavior When Switching Languages  
    1. Select a language in SettingsForm  
    2. Save to SettingsManager  
    3. LangManager.Load() will be called the next time the app is launched.  
    4. The UI will be displayed in the new language.

### 10.7 Design Features  
    Drive Indicator AI's multilingual support excels in the following ways :  
      1. JSON-based and highly scalable  
        No code changes are required to add new languages.  
      2. Robust against missing translations  
        If a key is not found, the key name is returned, preventing UI corruption.  
      3. Language lists are also managed using JSON.  
        UI and logic are completely separated.  
      4. Language is stored in a settings file.  
        The user's selected language is reflected the next time the application is launched.  
      5. Easy UI redraw  
        Bulk UI updates are performed with SettingsForm.ApplyLanguage().
